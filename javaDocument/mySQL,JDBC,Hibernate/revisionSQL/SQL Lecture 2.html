Creating Tables in MySQL
Creating a Table using the Create Table Syntax:
Once we have created a database in our MySQL Database Instance, it is quite easy to create tables.
Although creating tables manually using the Create Table syntax can become quite a laborious task if the data that we have at hand is quite huge.
The Create Table syntax has the following arguments to consider while creating a table in MySQL:
Name of the table
Names of columns
Datatypes of the columns
Constraints on the columns (Optional) - More in detail on this in advanced SQL module.
Create Table Syntax:
Create Table Customers ( Column1 datatype Primary Key NOT NULL, Column1 datatype constraint if any, . . . Column_n datatype, Primary Key (One or more columns));
SQL
Example Syntax:
Create Table Customers ( Customer_ID int Primary Key NOT NULL, First_Name varchar(20), Last_Name varchar(20), Phone_Number int, Address Varchar(100));
SQL
Understanding Datatypes:
Imagine you are creating a table in your database and all of its columns are of the type text. Which essentially means that your data be it words of English or integers or numbers etc., all are of the type “text”. To elaborate the data might look like words and mathematical numbers but are just plain simple text and do not carry the properties of numbers which for example get added to each other if we put a plus sign between them. Now suppose we want to perform a sum operation on our data to calculate the total amount that was spent by our customers in a particular day, how are we going to make this possible?
This is where the need for having datatypes comes in. If we specifically assign the type int for integer to a column full of numbers, it becomes easy for us as users and analysts to play with these numbers and get the desired output from them be it total amount spent, average amount spent etc.
Datatypes in SQL:
We usually deal with numbers, combinations of alphabets called characters or string and date time.
As we deep dive into data handling and querying with SQL, we will explore the different datatypes that exist within the above three categories.
Understanding Constraints:
Imagine you are creating a table manually in your MySQL Database.
You have defined the column names and are ready to fill in the data.
You have about 100 rows of data, you are working late at night in the office, you need to go home.
You decide to fill in 50 rows now and 50 rows later when you come back the next day.
The next day you come back, fill in data of the remaining 50 rows and finish the work.
You then submit this data to your teammate for certain analysis.
Your teammate then comes and reports that you had filled in text data into a column which was named salary and the salary data had been inserted into the address column for a few rows due to carelessness.
What do you do to avoid such things to happen in the first place?
Here is where SQL Constraints come into play.
SQL constraints are used to specify rules for the data in a table.
Constraints are used to limit the type of data that can go into a table.
This ensures the accuracy and reliability of the data in the table.
If there is any violation between the constraint and the data action, the action is aborted.
Different SQL Constraints and their functioning:
Primary key is the Unique identifier of rows in your tables
Foreign key is a constraint which defines how two tables are connected to each other
Check constraint helps us validate the data according to a few set conditions that we want our data to follow while inserting into our tables
Default constraint helps us assign a default value to all records of a particular column
Not Null constraint helps us avoid having null values in that particular column
Inserting Data into Tables
The INSERT statement allows you to:
Insert a single row into a table
Insert multiple rows into a table
Copy rows from a table to another table.
A. Inserting Single Row at a time:
Method 1 - Specifying column names and then inserting corresponding values:
Syntax:
INSERT INTO table1 (column1, column2, ...) VALUES (value1, value2,...);
SQL
Method 2 - Directly inserting values without specifying any column names:
Note: As many values as the number of columns need to be inserted for each row.
Here, The order of column values should match with order of columns in Schema
Syntax:
INSERT INTO table1 VALUES (value1, value2,...);
SQL
Method 3 - Entering values for only a select columns:
Syntax:
INSERT INTO table1 (column1, column3) VALUES (value1, value3);
SQL
B. Inserting multiple rows at a time:
INSERT INTO table1 VALUES (value1, value2,...), (value1, value2,...), (value1, value2,...), ...;
SQL
C. Copying rows from other tables:
SELECT * INTO Table2 FROM Table1 WHERE condition;
SQL
Reading Data in MySQL
SELECT statement is used to print/ask the MySQL Server to show a table as an output of the data that we want to read.
The SELECT statement lets you choose the columns out of the table that you want to visualize.
The SELECT statement consists of 4 parts:
Keyword: SELECT
Names of columns you want to print (* or specific column names)
Keyword: FROM
Name of table the columns belong to
We will look into printing columns from multiple tables when we go through the concept of joins later in the course.
You can either print all the columns present your table all at once by SELECTing * FROM the table.
Syntax:
SELECT * FROM hacker;
SQL
Output:
The “*” (Star) Symbol here represents all the columns of your table.
You can also choose to just print the columns that you like by just typing their names in the SELECT statement.
Syntax:
SELECT name, score from hacker;
SQL
Output:
There are many other clauses that combine with the SELECT statement to customize the kind of data that we want the database for us to throw back.
One such example is the WHERE Clause. You can use the WHERE Clause to filter the records according to your specific needs.
Syntax:
SELECT name, score from hacker WHERE score < 100;

Updating Data in SQL
Update statement is used to change data in existing tables.
In the update statement, Indicate table_name in which you want to update.
Specify the Columns and the values that you want to update.
Specify the condition in WHERE for rows which needs to be updated
General Syntax:
UPDATE table_name SET column1 = value1, column2 = value2 WHERE condition;
SQL
In general the UPDATE statement consists of 6 parts:
Keyword: UPDATE
Name of the table in which data change needs to take place.
Keyword: SET
Specifying the new value that a particular column’s fields need to change to.
Keyword: WHERE
Condition to filter the rows in which the new value needs to gets filled in, replacing the old value.
Reading the table:
Syntax:
SELECT * FROM hacker;
SQL
Output:
If you want to update all ranks in the rank column to a certain value, lets say 1, we execute the following:
Syntax:
UPDATE hacker SET hacker.rank = 1;
SQL
Output:
Reading the updated data:
Syntax:
SELECT * FROM hacker;
SQL
Output:
The above syntax updated the rows of the entire table in the rank column.
If you want to update the values of some specific fields for example the fields in the country column which depict “USA” to “United States”, the following query needs to be executed, here we introduce the WHERE Clause.
Syntax:
UPDATE hacker SET Country = 'United States' WHERE Country = 'USA';
SQL
Output:
Now reading the updated data:
Syntax - the same SELECT statement to be used:
SELECT * FROM hacker;
SQL
Output - Values in Country column previously having “USA” now contain the value “United States”:
Missing to put the where condition will update the rows of the entire table! Hence you need to be very careful when working with this dangerous query with tables having millions of rows.
Deleting Data in SQL
Delete statement is used to remove one or more rows from the table
Indicate table_name in DELETE statement from which you want to remove the rows
Specify the condition in WHERE for rows which needs to be deleted.
If you want to delete all the rows, omit the WHERE condition here
Syntax:
DELETE FROM table_name WHERE condition;
SQL
In general, the DELETE statement consists of the following parts:
Keyword: DELETE
Name of the table whose rows are to be deleted.
Keyword: WHERE
Condition to apply in the WHERE Clause to filter on the exact rows which need to be deleted.
Unlike updating single fields at a time in UPDATE statement, the DELETE statement deletes the entire row from the table.
Reading the table:
Syntax:
SELECT * FROM hacker;
SQL
Output:
Suppose we want to delete the record of the person whose username is “tarunmer” we execute the following:
Syntax:
DELETE FROM hacker WHERE name = 'tarunmer';
